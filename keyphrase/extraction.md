# キーフレーズ抽出（Keyphrase Extraction）

> 参考: https://www.m3tech.blog/entry/2021/12/23/110000

---

## 第1章 キーフレーズ抽出とは

### 1.1 定義

**キーフレーズ抽出**とは、文書中からその文書をよく表現する「句（フレーズ）」を自動的に取り出す技術である。

- 抽出されるフレーズは **名詞句** が中心
- 人手でタグ付けすることなく、文書の主要トピックを把握できる

### 1.2 活用用途

| 用途 | 説明 |
|------|------|
| 要約 | 文書の重要語を一覧で提示 |
| タグ付け | 文書への自動ラベル付与 |
| 検索 | キーフレーズを検索インデックスとして利用 |
| 類似文書検索 | キーフレーズ同士の類似度で関連文書を発見 |

## 第2章 手法の概要

### 2.1 採用手法

エムスリー社の実験では、EMNLP 2021 採択論文：

> **"Unsupervised Keyphrase Extraction by Jointly Modeling Local and Global Context"**

で提案された手法を採用した。

### 2.2 手法の特徴

この手法は **教師なし学習（Unsupervised）** であり、ラベル付きの学習データを必要としない。
BERT などの **事前学習済み言語モデル** のみで動作する。

### 2.3 2つのアプローチの統合

手法は以下の2つの観点を組み合わせている：

#### ① Embedding-based（埋め込みベース）
- 文書全体のベクトル表現と各フレーズのベクトル表現を比較
- 文書の意味に最も近いフレーズを重要語として抽出
- **Local Context（局所文脈）** を活用

#### ② Graph-based（グラフベース）
- フレーズの共起関係をグラフ（ネットワーク）として表現
- グラフ上で中心性の高いフレーズを重要語として選択
- **Global Context（大局的文脈）** を活用

```
文書
 ├─ Embedding-based → フレーズと文書の意味的近さ
 └─ Graph-based     → フレーズ同士の共起ネットワーク中心性
              ↓
         両者を統合してキーフレーズを選出
```

## 第3章 実験と結果

### 3.1 テックブログへの適用

**入力**: エンジニアリングブログの記事テキスト

**抽出結果（良好例）**:
- 「ABテスト」
- 「ユーザー内相関」

**課題（ノイズ例）**:
- 「私自身」のような内容に無関係な句も抽出される場合がある

### 3.2 医療相談サービス（AskDoctors）への適用

**入力**: 患者からの医療相談テキスト

**抽出結果（良好例）**:
- 「関節痛」
- 「筋肉痛」

**課題**:
- 同格で列挙された複数の症状が、すべて均等に抽出されるわけではない

### 3.3 類似文書検索への応用実験

キーフレーズを使った類似文書検索と、他の手法を比較した結果：

| 手法 | 特徴 |
|------|------|
| 全単語 | 一般的な語も含み、ノイズが多い |
| 名詞句のみ | 絞り込みはされるが、トピック感度がやや低い |
| **キーフレーズのみ** | 「監視」「SLO」などトピックに敏感な検索が可能 |

→ キーフレーズを使うと **トピックに特化した検索** が実現しやすい

## 第4章 課題と展望

### 4.1 現状の課題

- 完全な実務運用レベルの精度にはまだ達していない
- ノイズ（不要なフレーズ）が混入することがある
- ドメインによって精度にばらつきがある

### 4.2 改善の方向性

- **ドメイン特化チューニング**: 医療・技術など特定領域のデータで調整する
- **後処理フィルタ**: ストップワードや短すぎる句を除去する
- **ユースケースに合わせた閾値調整**: 抽出数・スコア閾値の最適化
