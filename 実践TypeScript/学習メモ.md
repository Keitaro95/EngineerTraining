useReducer
useStateの強化バージョン。更新関数の中身の引数に動きをつけられる。
この更新関数の中の引数関数は別に書いてもいい。
だから疎結合にできる。
```tsx
const [rstate, dispatch] = useReducer(prev => ++prev, 0);
const [rstate, dispatch] = useReducer((prev, { type }) => {
    switch (type) {
        case "+":
            return ++prev;
        case "-":
            return --prev;
        default:
            throw new Error()
    }
}, 0)
const rcountUp = () => {
    dispatch({ type: "+" })
}
const rcountown = () => {
    dispatch({ type: "-" })
}
```

useContext
コンポーネントを跨いでcontextを渡せる。
すなわち、コンポーネントを跨いでも、値を共有できるから便利。
```tsx
親コンポーネント
export const MyContext = createContext("hello")

孫コンポーネント
const value = useContext(MyContext)
```
useContextとstateの連携
まずコンテキストを作成
export const MyContext = createContext()

コンポーネント関数の中で
const stateAndSetter = useState(0)

return (
    // ここが大事。MyContextタグ以下に書かれた全ての子孫にコンテクストを渡せるようにしてる。stateAndSetter(配列)は常に子孫からstateの値を受け取る。
    <MyContext.Provider value={stateAndSetter}>
        以下、子孫でuseStateを使って値が更新されると、stateAndSetterが更新されて、コンテクストが全員に伝わる。
        ```
        const [count, setCount] = useContext(MyContext);
        return (
            <div>
            <p>Child: {count}</p>
            <button onClick={() => setCount(count + 1)}>＋</button>

        例えば子孫コンポーネントでこういうことが可能
        ```
        const [count] = useContext(MyContext);
        return <p>OtherChild sees: {count}</p>;

useEffectの
