# テキスト正規化モジュール

## 概要

日本語テキストの前処理として、ひらがな数字・カタカナ数字・漢数字・全角数字を半角数字に変換します。これにより、Presidioの既存パターンマッチング精度が向上します。

**Ginza不要**: このアプローチでは正規表現のみを使用するため、GinzaのNERに依存しません。

## 主要機能

### 1. ひらがな・カタカナ数字 → 半角数字変換

**特徴**: 2個以上連続する数字トークンのみ変換（誤変換防止）

```python
from text_normalizer import normalize_japanese_text

# 連続する数字トークンは変換される
normalize_japanese_text("ゼロキューゼロ いちにさんよん")
# → "090 1234"

# 孤立した数字トークンは変換されない（助詞の誤変換を防ぐ）
normalize_japanese_text("ご連絡ください")
# → "ご連絡ください" (そのまま)

normalize_japanese_text("にほんご")
# → "にほんご" (そのまま)
```

### 2. 全角数字 → 半角数字変換

NFKC正規化により自動的に変換されます。

```python
normalize_japanese_text("電話は０３−１２３４−５６７８です")
# → "電話は03−1234−5678です"
```

### 3. 漢数字 → 半角数字変換

基本数字（一二三四五六七八九）を半角数字に変換します。

```python
normalize_japanese_text("口座番号は普通一二三四五六七")
# → "口座番号は普通1234567"

normalize_japanese_text("電話番号は〇九〇-いちにさん-よんごろく")
# → "電話番号は090-123-456"
```

## アルゴリズム

### ひらがな・カタカナ数字変換の仕組み

1. **数字トークンの定義**
   - ひらがな: `ぜろ`, `いち`, `に`, `さん`, `よん`, `ご`, `ろく`, `なな`, `はち`, `きゅう`, `く`, `し`, `しち`
   - カタカナ: `ゼロ`, `イチ`, `ニ`, `サン`, `ヨン`, `ゴ`, `ロク`, `ナナ`, `ハチ`, `キュー`, `キュウ`, `ク`, `シ`, `シチ`

2. **連続性の判定**
   - 正規表現で「数字トークンまたは数字・ハイフン・記号」を含む連続した文字列を「単語」として抽出
   - 各単語内で2個以上連続する数字トークンの塊のみを検出して変換
   - 孤立した数字トークン（1個だけ）は変換しない

3. **例**
   ```
   入力: "ぜろさんに、いちにさんよん、ごろくです"
   
   ステップ1: 単語抽出
   - "ぜろさんに" (数字トークン3個連続)
   - "いちにさんよん" (数字トークン4個連続)
   - "ごろく" (数字トークン2個連続)
   
   ステップ2: 変換
   - "ぜろさんに" → "032"
   - "いちにさんよん" → "1234"
   - "ごろく" → "56"
   
   出力: "032、1234、56です"
   ```

## テストケース

実行例:


### カバーしているケース

| 入力 | 出力 | 説明 |
|------|------|------|
| `ゼロキューゼロ いちにさんよん` | `090 1234` | カタカナ数字の連続 |
| `ご連絡ください` | `ご連絡ください` | 孤立した「ご」は変換しない |
| `にほんご` | `にほんご` | 孤立した「に」は変換しない |
| `ぜろさんに-いちにさんよん-ごろく` | `032-1234-56` | ハイフン区切り |
| `電話は０３−１２３４−５６７８です` | `電話は03−1234−5678です` | 全角→半角 |
| `口座番号は普通一二三四五六七` | `口座番号は普通1234567` | 漢数字 |
| `03-いちにさんよん-5678` | `03-1234-5678` | 混在ケース |

## Presidio統合

`sample_usage.py` での使用例:

```python
from text_normalizer import normalize_japanese_text

# テキストを正規化
text_normalized = normalize_japanese_text(text)

# Presidioで検出
results = analyzer.analyze(text=text_normalized, language="ja")
```

## 利点

1. **誤変換の防止**: 孤立した数字トークン（助詞など）は変換されない
2. **高精度**: 2個以上連続する塊のみを対象とすることで、文脈に依存しない変換が可能
3. **軽量**: 正規表現のみで実装、Ginzaなどの重いライブラリ不要
4. **汎用性**: 電話番号、マイナンバー、口座番号などあらゆる数字表記に対応

## 制限事項

- 「十」「百」「千」などの位の数（じゅう、ひゃく、せん）は別途処理が必要
- 年月日の文脈では `kanji-to-number` ライブラリが必要（オプション）
